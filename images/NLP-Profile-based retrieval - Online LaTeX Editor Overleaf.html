3.1 Description of the dataset
Besides the basic idea of developing an application where users of different profiles
can access various articles of different sources, for implementation purposes we used
a rather simplified dataset available online. Our objective is to highlight the applied
document classification techniques which later can be generalized to address more
complicated applications.
The open-source BBC dataset[1] contains 2,225 articles from BBC news website
corresponding to stories in five topical areas from years 2004-2005. Each article is
labeled with one of the following five classes:
•
sport
•
business
•
politics

4.1 Data processing
There is no need to have a deep knowledge about how machine learning models
learn from data to understand that machine learning algorithms do not understand
letters, words or sentences. A transformation from words, sentences or documents
into numerical vector has to been done and when dealing with language is very
important to homogenize words that have the same meaning between all documents.
That is why, the following steps are systematically apply to the raw text of each
article of our dataset:
1.
To lowercase:
Converting each word of the article to lower case.
2.
Remove punctuation:
Punctuation does not add any significant value to the
meaning of the document by Natural Language Processing (NLP) standards.
3.
Remove stop words:
Words like ’a’, ’about’, ’after’ are some the words
defined as stop words and are very common and repetitive words present in
each document that do not have any information significance to characterize
a text because every document has large number of them.
4.
Stemming:
Is the process of obtaining the root of the word. This way we
obtained the same representation of derive words and reduce the complexity
of the articles.

4.2 Embedding techniques
Text classification can be done in two different ways: manual and automatic clas-
sification. In the former, a human annotator interprets the content of text and
categorizes it accordingly. This method usually can provide quality results but it’s
time-consuming and expensive. The latter applies machine learning, natural lan-
guage processing, and other techniques to automatically classify text in a faster and
more cost-effective way.
Figure 4.2: Document Classification
In this project we work only with automatic classification. The first step towards
training a classifier with machine learning is feature extraction. Machine learning
algorithms are incapable of learning from raw text so we have to use methods to
transform each text into a numerical representation in the form of a vector. This
process is called Word Embedding and in the following lines we will present some
of the different types of embedding alongside their advantages and disadvantages of
the rest.
4.2.1 Count Vector
Consider a Corpus C of D documents
{
d1,d2,...,dD
}
and N unique tokens extracted
out of the corpus C. The N tokens will form a dictionary and the size of the Count
Vector matrix M will be given by D X N. Each row in the matrix M contains the
frequency of tokens in document D(i).
In real world applications we might have a corpus which contains millions of
documents. And with millions of document, we can extract hundreds of millions
of unique words. So basically, the matrix that will be prepared like above will be
a very sparse one and inefficient for any computation. So an alternative to using
every unique word as a dictionary element would be to pick a top number of words
based on frequency and then prepare a dictionary.
